# retired_skills_manifest.txt — ownership data for aitask_prune_retired_skills.sh
#
# Retired skill surfaces that installed projects must stop carrying, plus the
# set of git blob hashes the framework ever SHIPPED at those paths.
#
# WHY HASHES: install_skills() / setup_codex() / setup_opencode() are additive
# `cp` loops — they never remove a wrapper that disappeared upstream, so an
# upgraded project keeps the retired files. But an exact path is NOT proof of
# framework ownership: a user may have customized the skill, kept their own
# unrelated skill at that name, or edited an untracked staging wrapper. The
# pruner therefore deletes a file ONLY when `git hash-object <file>` appears in
# the SHA set below; anything else is preserved and reported. Release tarballs
# copy files verbatim and this repo has no .gitattributes (no CRLF/text filter),
# so an installed copy hashes identically to the blob it came from.
#
# The SHA set is FLAT (not keyed by path) on purpose: it also covers the staging
# copies under aitasks/metadata/{codex,opencode}_skills/, which are byte-for-byte
# `cp`s of the agent-tree wrappers.
#
# RECORD TYPES (tab-separated):
#   DIR <path>    retired directory   — removed all-or-nothing (every file must
#                                       match a SHA and no extra file present)
#   FILE <path>   retired single file — removed when its hash matches
#   STEM <name>   rendered-closure stem — matched EXACTLY against
#                 <root>/<stem>-<profile>[-<agent>]- dirs. Never prefix-globbed:
#                 the required hyphen after the stem is what stops a retired
#                 stem that is a PREFIX of a live skill from swallowing that
#                 skill's renders (retiring `aitask-pick` must not delete
#                 `aitask-pickn-fast-`).
#                 Closures are NEVER removed by an upgrade (their content is a
#                 function of the user's own profiles, so ownership cannot be
#                 proven); they are reported, and removed only under the
#                 explicit --prune-rendered flag.
#
# REGENERATE (run from the repo root; the prefixes are the authoring surfaces
# that were deleted — history still holds every version they shipped in):
#
#   PFX=".claude/skills/aitask-pickn .claude/skills/task-workflown \
#        .agents/skills/aitask-pickn .opencode/skills/aitask-pickn \
#        .opencode/commands/aitask-pickn.md"
#   git log --all --pretty=format:%H -- $PFX | while read -r c; do
#       git ls-tree -r "$c" -- $PFX 2>/dev/null | awk '{print $3}'
#   done | sort -u
#
