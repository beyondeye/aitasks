## Probable User Intent

The user noticed that running `/aitask-explore` did not produce the first
interactive prompt immediately — the `git pull --ff-only` (and stale-lock
cleanup) in Step 0 had to complete first. They reasoned that because task
creation happens at the very end of the flow, deferring the sync to a point
"just before checking for similar tasks already existing" would cut the time
to the first `AskUserQuestion` without losing the correctness benefit.

The change is intentionally scoped to `aitask-explore` only. The sibling
skills (`aitask-fold`, `aitask-review`, `aitask-pr-import`) also sync upfront
but were left untouched in this task; `aitask-pick` was identified as a case
where the upfront sync is genuinely load-bearing (it feeds the first task
