## Context

Every previously-templated skill has a vestigial pre-templating-era "Source of Truth" wrapper at `.opencode/skills/<skill>/SKILL.md` that points to `.claude/skills/<skill>/SKILL.md`. That target has since become the §3b Claude stub (not a workflow), so following the pointer routes OpenCode through the wrong agent root.

pickweb (t777_15) already corrected its own case. This task generalizes the fix to the remaining 8 templated skills, patches the wrapper auditor so future regenerations emit the correct §3d-style stub for templated skills, and removes hardcoded per-skill switches from the verifier.

## Approach

### 1. Rewrite 8 OpenCode skill files

Replace `.opencode/skills/<skill>/SKILL.md` for each of:
- aitask-pick, aitask-explore, aitask-review, aitask-fold, aitask-qa, aitask-pr-import, aitask-revert
- aitask-pickrem (uses conditional-Read pattern, headless prerendered)

Each new file mirrors `.opencode/commands/<skill>.md` byte-for-byte in the body, with skill frontmatter added (`name:` required by OpenCode skills, plus `description:` copied from the command file).

**Plain §3d pattern** (7 skills: pick, explore, review, fold, qa, pr-import, revert):
```markdown
---
name: <skill>
description: <copied from .opencode/commands/<skill>.md>
---

@.opencode/skills/opencode_planmode_prereqs.md
@.opencode/skills/opencode_tool_mapping.md

This is a profile-aware skill stub. Execute these steps in order, then stop:

1. **Resolve active profile.** Parse $ARGUMENTS for `--profile <name>`.
   If found, use that as `<profile>` and remove the `--profile <name>`
   pair. Otherwise run:
   `./.aitask-scripts/aitask_skill_resolve_profile.sh <short>`
   and use the single-line stdout as `<profile>`.

2. **Render per-profile variant.** Run:
   `./.aitask-scripts/aitask_skill_render.sh <skill> --profile <profile> --agent opencode`

3. **Dispatch via Read-and-follow.** Read the file at
   `.opencode/skills/<skill>-<profile>-/SKILL.md` and execute its
   instructions as if they were this command, forwarding the (possibly
   stripped) $ARGUMENTS unchanged.
```

**Conditional-Read pattern** (pickrem only — pickweb already fixed in t777_15):
The "Pre-rendered variants for headless profiles…" preamble plus a "Render only if needed" Step 2 that short-circuits when the committed pre-rendered file exists. Mirror `.opencode/commands/aitask-pickrem.md` verbatim, with `---\nname: aitask-pickrem\ndescription: …\n---` frontmatter prepended.

### 2. Patch `aitask_audit_wrappers.sh::render_opencode_skill`

Detect templated skills via the presence of `.claude/skills/<skill>/SKILL.md.j2`. For templated skills, emit the §3d-style stub (matching what we wrote above). For non-templated skills, keep the legacy "Source of Truth" pointer.

Apply the same templated-vs-legacy detection symmetrically in `render_agents_skill` (lower-stakes — Codex `.agents/skills/<skill>/SKILL.md` is already the correct §3b stub, but a future `--force` regenerate would otherwise produce the wrong body).

The §3d stub generated by the auditor for templated skills uses the **plain** form (no conditional-Read). pickrem/pickweb's conditional-Read pattern is hand-authored; if either is ever deleted and regenerated by `apply-wrapper --force`, the conditional-Read line will need a manual touch-up. This is acceptable because (a) files are committed and (b) the regression test in step 4 will fail-loud on the missing "Render only if needed" if that path is exercised in pickrem (no test currently covers that property for the non-conditional 7).

Resolver key for the generated stub: `${skill#aitask-}` (matches every existing templated skill — see step 3 for why this is also the verifier default).

### 3. Simplify `aitask_skill_verify.sh::_resolver_key_for`

Every templated skill's resolver key today is `${skill#aitask-}` (verified across all 9 cases in the current switch). Replace the case-statement with the default convention and an optional sidecar override:

```bash
_resolver_key_for() {
    local skill="$1"
    local sidecar=".claude/skills/${skill}/resolver_key.txt"
    if [[ -f "$sidecar" ]]; then
        head -n1 "$sidecar"
    else
        echo "${skill#aitask-}"
    fi
}
```

Removes the per-skill switch arms and both TODOs at lines 78–79. No sidecar files are added (none needed — all current resolver keys match the default). The sidecar mechanism remains documented as the escape hatch for any future skill whose resolver key diverges.

### 4. Add regression test `tests/test_opencode_skill_legacy_pointers.sh`

Iterates over every templated skill (those with a `.claude/skills/<skill>/SKILL.md.j2`) and asserts, for `.opencode/skills/<skill>/SKILL.md`:
- File exists
- Body does NOT contain the literal "Source of Truth" phrase
- Body contains `aitask_skill_resolve_profile.sh <short>` where `<short> = ${skill#aitask-}`
- Body contains `aitask_skill_render.sh <skill>`
- Body contains `--agent opencode`
- Body contains `Dispatch via Read-and-follow`

Use the same `assert_*` helpers as `tests/test_skill_render_aitask_pickweb.sh` (copy/paste — no shared harness today). Include pickweb in the iteration so the 9th case is also asserted (matches Test 9 of the pickweb test).
