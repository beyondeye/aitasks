# Plan — t1071_4: Port `aitask-learn-skill` to Codex + OpenCode trees

## Context

`aitask-learn-skill` (Capability B of t1071) landed in the **Claude** tree in
t1071_2: a static, user-invocable `/learn`-style command that learns a skill from
a local file / URL / repo file-or-dir / tmux pane id, with a shared `generate.md`
core. Per CLAUDE.md ("done in the Claude Code version first … suggest separate
aitasks to update the corresponding skills/commands in the other supported coding
agents"), this task ports it to the two other supported agents — **Codex CLI**
(`.agents/skills/`) and **OpenCode** (`.opencode/skills/` + `.opencode/commands/`).

The framework's established mechanism for porting a **static** (non-`.j2`) skill is
a **thin wrapper**: a one-screen `SKILL.md` in each agent tree whose body redirects
the agent to the Claude source-of-truth `SKILL.md` and the relevant tool-mapping
file. The other agent then reads-and-follows the Claude `SKILL.md`, which in turn
reads the Claude-tree `generate.md` directly. No logic is duplicated.

## Scope decision — do NOT copy `generate.md` (explicit AC deviation)

The task's AC and the t1071_2 plan say to port "`SKILL.md` **+ `generate.md`**" to
each tree. Investigation shows this is the wrong shape for a wrapper port, so I am
**deviating explicitly** (not silently) and will update the task AC before
implementing:

- Hand-authored wrapper skills are **SKILL.md-only** in the Codex/OpenCode trees.
  Confirmed on disk:
  - `aitask-reviewguide-import` — `.agents/skills/.../SKILL.md` and
    `.opencode/skills/.../SKILL.md` only (no sub-procedure copies), even though it
    is the named reference model in this task.
  - `aitask-shadow` — its five `plan-*.md` sub-procedures live **only** in the
    Claude tree; the Codex/OpenCode copies are bare `SKILL.md` wrappers redirecting
    there.
- The sub-procedure `.md` files that *are* present under `.agents/skills/` /
  `.opencode/skills/` belong exclusively to **`.j2`-rendered** skills
  (`task-workflow-*`, `aitask-qa-*`), auto-generated by the rerender driver — not
  to hand-authored static wrappers.
- The wrapper redirect fully covers `generate.md`: the Codex/OpenCode agent follows
  the Claude `SKILL.md`, whose Step 3 says "Read and follow `generate.md`" (a
  same-dir relative reference → `.claude/skills/aitask-learn-skill/generate.md`,
  which exists). Copying `generate.md` into the other trees would create a
  divergent duplicate that drifts from the source — a maintenance defect, not a
  feature.

**Conclusion:** port **SKILL.md wrappers only** (Codex skill, OpenCode skill,
OpenCode command stub). This matches `project_shadow_skill_wrapper_no_port` and
`project_closure_changes_autorender_no_port`.
