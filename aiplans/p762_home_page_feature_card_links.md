# Plan for t762: Home page feature card links

## Context

The website home page (`website/content/_index.md`) opens with three prominent feature cards (Agentic IDE, Long-term memory, Tight git coupling) but none of them are clickable. Visitors who scan the cards have no way to drill into the corresponding documentation. Three small edits make the cards link out to the right doc pages, with the first card linking in-page to the existing tour mosaic, and a "See all TUIs →" CTA below the mosaic to give the tour section a clear exit ramp into the full TUI docs.

## Implementation steps

### Step 1 — Add anchor on the tour mosaic heading

Currently lines 52-56:
```html
{{< blocks/section color="white" >}}
<div class="text-center mb-4">
<h2>🎛️ Take the tour</h2>
<p>Five TUIs share a single tmux session. Click any of them to dive in.</p>
</div>
```

Change `<h2>🎛️ Take the tour</h2>` to `<h2 id="take-the-tour">🎛️ Take the tour</h2>`. The heading is already raw HTML inside a Hugo `blocks/section`, so adding the `id` attribute is straightforward and doesn't depend on Hugo's auto-generated anchors.

### Step 2 — Add `url` to each of the 3 top feature cards

Lines 22-32, three `{{% blocks/feature %}}` blocks. Add a `url=` parameter to each opening tag, matching the existing pattern at lines 117-127 (e.g. `url="docs/installation/"`).

| Card (current line) | Add `url=` value |
|------|-------------|
| `Agentic IDE in your terminal` (line 22) | `"#take-the-tour"` |
| `Long-term memory for agents` (line 26) | `"/docs/concepts/agent-memory/"` |
| `Tight git coupling, AI-enhanced` (line 30) | `"/docs/workflows/#git"` |

Note on URL style: existing cards on lines 117-127 use no leading slash (`url="docs/installation/"`). The task description specifies leading-slash absolute paths (`/docs/concepts/agent-memory/`, `/docs/workflows/#git`). Hugo resolves both forms correctly; following the task description's exact URL strings to be conservative. If the live preview reveals broken links, fall back to the no-leading-slash form to match the existing pattern.

The first card uses an in-page fragment `#take-the-tour`; bare fragments resolve to the current page, so it scrolls to the tour mosaic without leaving the home page.
