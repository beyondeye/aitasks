## Context

The `aidocs/` directory has extraction scripts for Codex CLI and Gemini CLI that invoke each CLI in batch mode to self-document their available tools. We need equivalent scripts for Claude Code and OpenCode to help when porting skills between code agents.

## Implementation

### 1. Create `aidocs/extract_claudecode_tools.sh` [done]

Same pattern as existing scripts. Uses Claude Code's `-p` (print mode) and `--dangerously-skip-permissions` flags for non-interactive execution.

### 2. Create `aidocs/extract_opencode_tools.sh` [done]

Uses OpenCode's `opencode run --dir` for non-interactive execution. OpenCode lacks a `--dangerously-skip-permissions` flag, so the script temporarily creates an `opencode.json` with `"permission": "allow"` in the project root, then restores the original config (if any) on exit via a trap.

### 3. Update prompts in all extraction scripts [done]

Added exclusion of skills/custom commands to all four scripts (codex, gemini, claude, opencode). Prompts now specify "built-in tools only" and explicitly exclude skills/user-defined extensions.

## Final Implementation Notes
- **Actual work done:** Created extraction scripts for Claude Code and OpenCode, plus updated all four extraction script prompts to exclude skills/custom commands
- **Deviations from plan:** OpenCode required a temporary config file workaround since it lacks a `--dangerously-skip-permissions` CLI flag (only has config-based `"permission": "allow"`)
- **Issues encountered:** OpenCode `run` mode auto-rejected `external_directory` permissions for its own install dir; resolved by temporarily writing `opencode.json` with `"permission": "allow"` and restoring via trap
- **Key decisions:** Used trap-based cleanup for OpenCode config to ensure restore even on script failure
