# Plan — Rename `impromptu` → `autonomous` (t849)

## Context

The recently-landed manual-verification auto-execution feature (t843)
introduced **`impromptu`** as a strategy / profile-value paired with
**`prebuilt`**. The term is too exotic for a value that both AI agents
and humans read in profile YAML, AskUserQuestion options, and procedure
prose. The intended meaning — *the agent is free to improvise per item,
executes autonomously, and only reports a-posteriori in the plan file* —
maps more cleanly to **`autonomous`** (confirmed by user). This task is
a verbatim rename across the source-of-truth skill files, plus
regeneration of the autogenerated rendered variants and golden tests
that diff against them.

No callers exist yet that read `manual_verification_auto_mode:
impromptu` from a committed profile YAML — `grep -rn "impromptu"
aitasks/metadata/profiles/` is empty — so no profile-YAML migration is
needed. t843 added the feature, but committed profiles still default to
`ask`.

## Regeneration steps (after source edits)

### 1. Re-render via the driver

```bash
for prof in default fast remote; do
  ./.aitask-scripts/aitask_skill_rerender.sh "$prof"
done
```

Each call emits `RERENDERED:<N> (skill,agent) pairs for profile '<name>'`. The driver iterates `claude codex opencode` and refreshes every `*-<profile>-` (and shared-root `*-<profile>-codex-`) directory. The renderer's per-file atomic overwrite means active agent sessions are safe.

### 2. Regenerate golden files

Three procedure goldens × three profiles = 9 files under `tests/golden/procs/task-workflow/`:

```bash

GOLDEN_DIR="tests/golden/procs/task-workflow"

for file in SKILL manual-verification auto-verification; do
  for prof in default fast remote; do

      "$WORKFLOW_DIR/${file}.md" "$PROFILES_DIR/${prof}.yaml" claude \
      > "$GOLDEN_DIR/${file}-${prof}.md"
  done
done
```

`profiles.md` has no golden (not in `WRAPPED_FILES_VARYING` in `tests/test_skill_render_task_workflow.sh`) — no golden update needed for it.
