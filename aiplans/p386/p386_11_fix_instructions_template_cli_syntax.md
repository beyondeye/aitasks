# Fix _instructions.md Template CLI Syntax

## Context

The `_instructions.md` template generated by `aitask_crew_addwork.sh` (lines 179-214) contains incorrect CLI syntax that doesn't match the actual Python/bash CLIs implemented in t386_2. Agents following these instructions would get command errors.

## Changes

**File:** `.aitask-scripts/aitask_crew_addwork.sh` lines 179-214

Six fixes in the heredoc template:

1. **Status set syntax:** `./ait crew status --crew ... --agent ... --set-status <status>` → `ait crew status --crew ... --agent ... set --status <status>` (subcommand before flag)
2. **Progress set syntax:** `./ait crew status --crew ... --agent ... --set-progress <N>` → `ait crew status --crew ... --agent ... set --progress <N>` (subcommand before flag)
3. **Heartbeat syntax:** `./ait crew status --crew ... --agent ... --heartbeat` → `ait crew status --crew ... --agent ... heartbeat` (subcommand, not flag)
4. **Command read syntax:** `./ait crew command --crew ... --agent ... --read` → `ait crew command list --crew ... --agent ...` (subcommand `list`, not `--read`)
5. **`./ait` → `ait`** throughout (use PATH-resolved command, not relative)
6. **Valid statuses:** `Waiting, Running, Completed, Aborted, Error` → `Running, Completed, Aborted, Error` (agents cannot set Waiting — it's the initial state set by the system)
