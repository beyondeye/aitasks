# t699 — Fix `.gitignore` trailing slashes that don't match symlinks

## Context

`setup_data_branch` in `.aitask-scripts/aitask_setup.sh:1170-1179` writes
`aitasks/` and `aiplans/` (with trailing slashes) into the project's
`.gitignore`. Trailing-slash gitignore patterns match **directories only**.
After the same function creates the symlinks `aitasks → .aitask-data/aitasks`
and `aiplans → .aitask-data/aiplans`, those symlinks are NOT matched by the
trailing-slash rules, so they appear as untracked in `git status` — producing
the "dirty working tree after `ait setup`" symptom (beyondeye/aitasks#13).

Reproduced just now in this repo: `git check-ignore aitasks aiplans` returns
nothing, and `git status --porcelain` shows `?? aitasks` / `?? aiplans`.

The fix is to drop the trailing slashes — `aitasks` (no slash) ignores both
the directory **and** the symlink. The task selects "Option 2": fix the seed
**and** migrate existing repos that already committed the trailing-slash form,
so the symptom doesn't linger forever for current users.

## Files to modify

1. `.aitask-scripts/aitask_setup.sh` — fix new-install seed + add migration
2. `tests/test_data_branch_setup.sh` — update existing assertions, add
   migration test, add clean-tree assertion
3. `.gitignore` (this repo) — apply the migration to the source repo so the
   `aitasks` / `aiplans` symlinks stop showing as untracked here

No `seed/` change is required: there is no `.gitignore` template under
`seed/` — the gitignore content is generated by `setup_data_branch` directly.

## Implementation

### 1. `.aitask-scripts/aitask_setup.sh` — `setup_data_branch` gitignore block

**Lines 1166-1179 today:**

```bash
# --- Step 7: Update .gitignore on main ---
local gitignore="$project_dir/.gitignore"
local gitignore_changed=false

if [[ ! -f "$gitignore" ]] || ! grep -qxF ".aitask-data/" "$gitignore" 2>/dev/null; then
    {
        echo ""
        echo "# Task data (lives on aitask-data branch, accessed via symlinks)"
        echo ".aitask-data/"
        echo "aitasks/"
        echo "aiplans/"
    } >> "$gitignore"
    gitignore_changed=true
fi
```

**Replace with:**

```bash
# --- Step 7: Update .gitignore on main ---
local gitignore="$project_dir/.gitignore"
local gitignore_changed=false

# Migration: rewrite legacy trailing-slash entries to symlink-safe form.
# Trailing-slash patterns (`aitasks/`) match only directories, missing the
# symlinks that setup_data_branch creates. Bare entries (`aitasks`) match
# both. Older repos may already have the legacy form committed.
if [[ -f "$gitignore" ]] && \
   { grep -qxF "aitasks/" "$gitignore" 2>/dev/null || \
     grep -qxF "aiplans/" "$gitignore" 2>/dev/null; }; then
    local tmp_gitignore
    tmp_gitignore="$(mktemp "${TMPDIR:-/tmp}/aitask_gitignore_XXXXXX")"
    awk '
        /^aitasks\/$/ { print "aitasks"; next }
        /^aiplans\/$/ { print "aiplans"; next }
        { print }
    ' "$gitignore" > "$tmp_gitignore" && mv "$tmp_gitignore" "$gitignore"
    gitignore_changed=true
fi

if [[ ! -f "$gitignore" ]] || ! grep -qxF ".aitask-data/" "$gitignore" 2>/dev/null; then
    {
        echo ""
        echo "# Task data (lives on aitask-data branch, accessed via symlinks)"
        echo ".aitask-data/"
        echo "aitasks"
        echo "aiplans"
    } >> "$gitignore"
    gitignore_changed=true
fi
```

Two changes in this hunk:

- **New install** (the existing append branch): `aitasks` / `aiplans` instead
  of `aitasks/` / `aiplans/`.
- **Existing repo** (new migration branch above): if the legacy
  trailing-slash entries are present, rewrite them in place. `awk` over a
  temp file is portable (avoids the GNU/BSD `sed -i` split that
  `aitask_setup.sh` does not currently address — `terminal_compat.sh`'s
  `sed_inplace` is not sourced here, so adding a sed dependency would mean
  also wiring up the source). The migration only touches exact-line matches
  (`^aitasks/$` / `^aiplans/$`); `aitasks/new/`, `aitasks/metadata/...`,
  etc. are untouched.

The existing `gitignore_changed=true` flow at lines 1184-1204 handles the
`git add .gitignore` + commit, so migration changes are committed alongside
any other gitignore edits in the same `ait: Configure task data branch …` /
`ait: Migrate task data to aitask-data branch` commit. No new commit message
is needed — both fresh-install and migration paths reach the same commit
block.

### 2. `tests/test_data_branch_setup.sh` — update existing tests + add new one

**Lines 177-178 today:**

`assert_file_contains` is `grep -qF` (substring), so the literal `aitasks/`
will no longer match a bare `aitasks` line. Tighten these to exact-line
matches against the symlink-safe form, and add the regression assertion the
task explicitly requests:

```bash

if grep -qxF "aitasks" "$TMPDIR_1/local/.gitignore"; then
    PASS=$((PASS + 1))

TOTAL=$((TOTAL + 1))
if grep -qxF "aiplans" "$TMPDIR_1/local/.gitignore"; then
    PASS=$((PASS + 1))

assert_file_contains ".gitignore has .aitask-data/" "$TMPDIR_1/local/.gitignore" ".aitask-data/"

# Regression: working tree must be clean after setup_data_branch — the

assert_eq "git status clean after setup_data_branch" "" "$porcelain"
```
