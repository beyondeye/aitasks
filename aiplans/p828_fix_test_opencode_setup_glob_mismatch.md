# Plan: Fix test_opencode_setup.sh glob-vs-count mismatch (t828)

## Context

`tests/test_opencode_setup.sh` Tests 1 and 2 fail because the test's
count side and loop side disagree on what counts as an "install-relevant"
skill.

Convention (memorised under `feedback_profile_variants_autogenerated`):
all `*-/` profile-variant dirs are autogenerated and not part of the
framework install distribution. Local installs only ship the tracked
`aitask-*/` stub skills. On dev machines, locally-rendered gitignored
variants (`aitask-pick-fast-/`, etc.) sit in the same directory and
pollute both sides of the test if they're read off disk.

The test today:
- Count: `find ... -name SKILL.md` → 53 (includes gitignored renders).
- Loop: `aitask-*/` on disk → 47 (still includes gitignored renders).

Both drift with local dev state. Both need to pin to the tracked install
set.

## Approach

Use `git ls-files '.opencode/skills/aitask-*/SKILL.md'` as the source of
truth for both the expected count and the packaging loop. This is the
**strict mirror** of what `aitask_setup.sh` actually packages (25 tracked
aitask-* dirs, including the 2 committed `aitask-*-remote-` exceptions).
