# Plan — t687: Auto-commit `.gitignore` after `setup_python_cache_gitignore`

## Context

After `ait setup` runs on a fresh clone of an aitasks-using repo, `git status`
shows `.gitignore` as modified-but-uncommitted because of the `__pycache__/`
rule the setup adds. The user is left with a dirty working tree they didn't
ask for.

`setup_python_cache_gitignore()` (`.aitask-scripts/aitask_setup.sh:1303-1326`)
intentionally does NOT commit. Its comment defers the commit to
`commit_framework_files()` at the end of setup. That deferred commit is
gated by an interactive `[Y/n]` prompt (`aitask_setup.sh:2552-2558`), so any
"no" answer (or stdin closure under some launchers) leaves the rule
applied-but-uncommitted. Reproduction confirmed that the dirty `.gitignore`
is exactly this case.

The user's suggested fix is to commit `.gitignore` immediately inside
`setup_python_cache_gitignore()`, matching the established pattern of
`setup_draft_directory()` (`.aitask-scripts/aitask_setup.sh:1291-1294`).
That pattern commits its own `.gitignore` edit unconditionally with a
descriptive `ait:` message — no approval gate, no dependency on a later
function — and is the right model here because the rule is part of the
user-asked-for setup and shouldn't require a second confirmation.

A related issue (trailing-slash `.gitignore` entries don't match the
`aitasks` / `aiplans` symlinks) is out of scope per the task description
and will be raised separately at archival time.

## Implementation

### Change 1 — `.aitask-scripts/aitask_setup.sh`

Replace `setup_python_cache_gitignore()` (currently lines 1299-1326) with a
version that commits its own change, mirroring `setup_draft_directory()`:

```bash
# --- Python cache artifact gitignore rule ---
# Runs after setup_draft_directory (which ensures .gitignore exists).
setup_python_cache_gitignore() {
    local project_dir="$SCRIPT_DIR/.."
    local gitignore="$project_dir/.gitignore"

    if [[ -f "$gitignore" ]] && grep -qxF "__pycache__/" "$gitignore"; then
        success "Python cache rule already in .gitignore"
        return
    fi

    info "Adding __pycache__/ to .gitignore (Python cache artifacts)..."

    if [[ -f "$gitignore" ]]; then
        echo "" >> "$gitignore"
        echo "# Python cache artifacts (generated by aitasks Python TUIs)" >> "$gitignore"
        echo "__pycache__/" >> "$gitignore"
    else
        {
            echo "# Python cache artifacts (generated by aitasks Python TUIs)"
            echo "__pycache__/"
        } > "$gitignore"
    fi

    # Commit the change if inside a git repo
    if git -C "$project_dir" rev-parse --is-inside-work-tree &>/dev/null; then
        (cd "$project_dir" && git add .gitignore && git commit -m "ait: Add __pycache__/ to .gitignore (Python cache artifacts)" 2>/dev/null) || true
    fi

    success "Python cache rule added to .gitignore"
}
```

Notable diffs vs. current:
- Drop the "Does NOT commit: ... commit_framework_files()" comment (lines 1301-1302)
- Add the `git add .gitignore && git commit` block before `success` (mirrors `setup_draft_directory:1291-1294`)
- Commit message: `ait: Add __pycache__/ to .gitignore (Python cache artifacts)` — parallels the draft-directory message

`commit_framework_files()` is unchanged — it remains the catch-all for any
other late-stage framework files. After this change, `.gitignore` simply
won't be in its modified-list anymore in the typical flow.

### Change 2 — `tests/test_setup_git.sh`

Append two new tests at the end (before the final summary print). They
follow the exact shape of Tests 7 and 8 (`test_setup_git.sh:222-260`):

- **Test: `setup_python_cache_gitignore` commits the change**
  1. `setup_fake_project`, `git init`, configure user
  2. Source the setup script (already sourced at line 90)
  3. Pre-create `.gitignore` with `aitasks/new/` and commit (clean baseline)
  4. Run `setup_python_cache_gitignore </dev/null >/dev/null 2>&1`
  5. Assert `.gitignore` contains `__pycache__/`
  6. Assert `git status --porcelain` is empty (the bug)
  7. Assert the last commit message contains
     `Add __pycache__/ to .gitignore`

- **Test: `setup_python_cache_gitignore` is idempotent**
  1. Same setup
  2. Run twice
  3. Assert `__pycache__/` appears exactly once
  4. Assert no second commit was created (commit count stable on second run)

These mirror existing Test 7 / Test 8 (`setup_draft_directory` ...)
verbatim except for the function name and the assertions.
