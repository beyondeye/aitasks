## Context

Foundation child for t749. The brainstorm TUI relies on
`br_groups.yaml` for the Status tab "Operation Groups" section, the
Recent Operations history in the Actions wizard, and (after t749) the
new operation badge / dashboard "Generated by" block / Operation
Detail screen. But the file is **only initialized empty** by
`init_session` and **never populated** when an operation runs ŌĆö so all
read sites currently see `{groups: {}}` in real sessions.

This child fills the gap: persist a group entry per operation at
registration time, with the existing spec fields only (no params
duplication ŌĆö see t749_2 for the reference primitive that lets the UI
pinpoint user inputs in their original on-disk locations).

## Implementation Plan

1. In `brainstorm_session.py`, add:
   ```python
   def record_operation(task_num, group_name, operation, agents,
                        head_at_creation):
       wt = crew_worktree(task_num)
       path = wt / GROUPS_FILE
       data = read_yaml(str(path)) or {"groups": {}}
       groups = data.setdefault("groups", {})
       groups[group_name] = {
           "operation": operation,
           "agents": list(agents),
           "status": "Waiting",
           "created_at": datetime.now().strftime("%Y-%m-%d %H:%M"),
           "head_at_creation": head_at_creation,
           "nodes_created": [],
       }
       write_yaml(str(path), data)

   def update_operation(task_num, group_name, **fields):
       wt = crew_worktree(task_num)
       path = wt / GROUPS_FILE
       data = read_yaml(str(path)) or {"groups": {}}
       grp = data.setdefault("groups", {}).get(group_name)
       if grp is None:
           return
       for k, v in fields.items():
           if k == "nodes_created" and isinstance(v, str):
               grp.setdefault("nodes_created", []).append(v)
           elif k == "agents_append" and isinstance(v, str):
               agents = grp.setdefault("agents", [])
               if v not in agents:
                   agents.append(v)
           else:
               grp[k] = v
       write_yaml(str(path), data)
   ```

2. In `init_session`, after `next_node_id(wt)` (line 152), call
   `record_operation(task_num, "bootstrap", "bootstrap", [],
   head_at_creation=None)` and append `n000_init` to its
   `nodes_created`.

3. In `apply_initializer_output` (line 338), after writing
   n000_init.yaml successfully, call
   `update_operation(task_num, "bootstrap",
   agents_append="initializer_bootstrap")`.

4. In `apply_patcher_output` (line 491), after `set_head(wt,
   new_node_id)` (line 588), derive the group name from
   `node_data["created_by_group"]` and call:
   ```python
   update_operation(task_num, node_data["created_by_group"],
                    nodes_created=new_node_id, status="Completed")
   ```

5. In `brainstorm_app.py:_run_design_op` (line 3814), capture the
   pre-registration HEAD and the group name, then on success call:
   ```python
   from brainstorm.brainstorm_session import record_operation
   record_operation(self.task_num, group_name, op, agents_list,
                    head_at_creation=current_head)
   ```
   For `explore`, `agents_list` is the full `agents` list. For
   `compare`/`hybridize`/`detail`/`patch`, it is `[agent]`.

6. (Out of scope of this child) For asynchronous explore/detail
   completion, the runner already updates per-agent `_status.yaml`. A
   later polish (in t749_5 or its post-review iteration) can add a
   poller in the OperationDetailScreen that re-derives the group's
   aggregated `status` from agent statuses on refresh. Do NOT add a
   poller in this child.

## Notes for Sibling Tasks

- `nodes_created` is the source of truth for "which nodes did this
  operation produce". Sibling tasks (t749_2..t749_6) should read it
  from `br_groups.yaml` rather than scanning all nodes for matching
  `created_by_group`.
- For legacy sessions whose `br_groups.yaml` has `groups: {}`, all
  later children must fall back gracefully (badge `?`, dashboard
  one-liner, OperationDetailScreen "no group recorded" message). Do
  not retroactively backfill ŌĆö that is a separate ait task if ever
  needed.
