Audit the active Claude Code memory store for this project and consolidate
durable engineering conventions into `aidocs/` (the on-demand authoring-reference
docs that CLAUDE.md points to). Trim memories that are already redundant with
existing aidocs. The audit found **no clearly obsolete memories** — every memory
still maps to live code or active work — so this task is promotion + redundancy
cleanup, not obsolescence deletion.

Memory store: `~/.claude/projects/-home-ddt-Work-aitasks/memory/` (index in
`MEMORY.md`). 26 content memories audited (excluding `MEMORY.md` and the 2
hardware-only notes `user_machine_omarchy_g16`, `project_g16_line_out_override`).

## Background / how this came up

Side-investigation also confirmed (no action needed, see "Out of scope" below):
`ait setup` already creates AGENTS.md **if missing** and populates it with the
shared aitasks instructions — it is NOT CLAUDE.md-only.
`setup_code_agents()` calls `update_agentsmd()` unconditionally
(`.aitask-scripts/aitask_setup.sh:1054`); the shared injector
`insert_aitasks_instructions()` (`:1005`) creates the file when absent
(`:1013-1015`) and otherwise replaces the `>>>aitasks`/`<<<aitasks` marker block.
AGENTS.md receives shared Layer-1 only; CLAUDE.md / `.codex/instructions.md` /
`.opencode/instructions.md` get Layer-1 + agent-specific Layer-2. Source of truth:
`seed/aitasks_agent_instructions.seed.md` + `seed/<agent>_instructions.seed.md`.

## Goal

For each promotion below: lift the memory's substance into the target aidoc
(extend an existing section or add a short new one, matching that doc's voice and
density). After the convention lives in aidocs, update the memory to a one-line
pointer to the aidoc (or delete it) and fix the `MEMORY.md` index line. Verify the
named files/functions still exist before citing them in aidocs.
