## `ait setup` modifies `.gitignore` but leaves the change uncommitted, putting the working tree in a dirty state

## Summary

After `ait setup` completes on a fresh clone, `git status` shows
`.gitignore` as modified-but-not-staged. The user is left in a dirty
working tree they didn't ask for, with no clear signal during setup that
the file was edited or that they should commit it.

## Reproduction (observed 2026-04-27)

1. `git clone <repo>` (a repo already using aitasks in data-branch mode).
2. `cd <repo> && ./ait setup`
3. `git status` after setup completes:

       Changes not staged for commit:
         modified:   .gitignore

The diff added a Python cache rule:

    +# Python cache artifacts (generated by aitasks Python TUIs)
    +__pycache__/

Setup output included a single line about it:

    [ait] Adding __pycache__/ to .gitignore (Python cache artifacts)...
    [ait] Python cache rule added to .gitignore

…but no instruction to commit, and no auto-commit.

## Why this is a problem

- A user who just cloned and ran setup may not realize their working tree
  is dirty before they start their first task.
- If the user runs a workflow that branches off `master`, the dirty
  `.gitignore` will follow into the new branch.
- It's surprising behavior — every other piece of state setup creates
  (the `aitask-data` worktree, the `aitask-ids` branch, `userconfig.yaml`)
  is either gitignored or pushed. Only `.gitignore` itself is left
  half-applied.

## Suggested fix

Auto-commit the `.gitignore` edit as part of setup (e.g. with a message
like `ait: Setup — add __pycache__/ to .gitignore`). This matches the
pattern already used for the `aitask-ids` and `aitask-data` branches,
which setup initializes and pushes on the user's behalf.

## Related (possibly worth a separate issue)

The `aitasks/` and `aiplans/` entries in the gitignore template use
trailing slashes (`aitasks/`, `aiplans/`). Trailing slash matches
directories but not symlinks, so after `ait setup` creates the symlinks
`aitasks -> .aitask-data/aitasks` and `aiplans -> .aitask-data/aiplans`,
`git status` lists them as untracked. Dropping the trailing slashes
(`aitasks`, `aiplans`) would ignore both directories and symlinks.
