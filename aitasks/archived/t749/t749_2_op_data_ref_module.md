## Context

Pure data-layer child for t749. Defines the typed reference primitive
(`OpDataRef`) that the dashboard "Generated by" block (t749_4) and the
Operation Detail screen (t749_5) use to pinpoint operation
inputs/outputs/logs in their original on-disk locations rather than
duplicating values into `br_groups.yaml`.

The user explicitly rejected duplicating params in `br_groups.yaml`
during plan review and asked for a "reference format that pinpoints
data in the original group operation input/outputs/definition" —
this child delivers exactly that.

## Key Files to Modify

- `.aitask-scripts/brainstorm/brainstorm_op_refs.py` — **NEW FILE**.
  Defines `OpDataRef`, `resolve_ref`, `file_for_ref`, and the
  `list_op_inputs` / `list_op_outputs` / `list_op_logs` /
  `list_op_definition` helpers.
- `.aitask-scripts/brainstorm/__init__.py` — re-export the public
  surface so consumers can `from brainstorm.brainstorm_op_refs import
  OpDataRef, list_op_inputs, ...`.

## Reference Files for Patterns

- `.aitask-scripts/brainstorm/brainstorm_dag.py` — pure functions over
  a session_path; mirror this style (no Textual imports).
- `.aitask-scripts/brainstorm/brainstorm_schemas.py` — uses
  `@dataclass`-free plain constants; for `OpDataRef` we use
  `@dataclass(frozen=True)` per the parent plan.

## Implementation Plan

1. Create `.aitask-scripts/brainstorm/brainstorm_op_refs.py`:

   ```python
   from __future__ import annotations
   from dataclasses import dataclass
   from pathlib import Path

   from .brainstorm_dag import NODES_DIR, PROPOSALS_DIR, PLANS_DIR

   # Section anchor inside _input.md per operation
   _OP_INPUT_SECTION = {
       "explore":   "Exploration Mandate",
       "compare":   "Comparison Request",
       "hybridize": "Merge Rules",
       "detail":    None,
       "patch":     "Patch Request",
       "bootstrap": "Mandate",
   }

   _VALID_KINDS = {
       "agent_input", "agent_output", "agent_log",
       "node_proposal", "node_plan", "node_metadata",
       "session_spec",
   }

   @dataclass(frozen=True)
   class OpDataRef:
       kind: str
       target: str
       section: str | None = None

       def __post_init__(self):
           if self.kind not in _VALID_KINDS:
               raise ValueError(f"OpDataRef: bad kind {self.kind!r}")

   def file_for_ref(session_path: Path, ref: OpDataRef) -> Path:
       if ref.kind == "agent_input":
           return session_path / f"{ref.target}_input.md"
       if ref.kind == "agent_output":
           return session_path / f"{ref.target}_output.md"
       if ref.kind == "agent_log":
           return session_path / f"{ref.target}_log.txt"
       if ref.kind == "node_proposal":
           return session_path / PROPOSALS_DIR / f"{ref.target}.md"
       if ref.kind == "node_plan":
           return session_path / PLANS_DIR / f"{ref.target}_plan.md"
       if ref.kind == "node_metadata":
           return session_path / NODES_DIR / f"{ref.target}.yaml"
       if ref.kind == "session_spec":
           return session_path / "br_session.yaml"
       raise ValueError(f"unhandled kind {ref.kind!r}")

   def resolve_ref(session_path: Path, ref: OpDataRef) -> str:
       p = file_for_ref(session_path, ref)
       if not p.is_file():
           return ""
       text = p.read_text(encoding="utf-8")
       if ref.section is None:
           return text
       return _extract_md_section(text, ref.section)

   def _extract_md_section(text: str, header: str) -> str:
       """Extract the markdown subsection beginning at `## <header>` until
       the next `## ` line or EOF. Returns empty string if not found."""
       lines = text.splitlines()
       start = None
       for i, ln in enumerate(lines):
           stripped = ln.lstrip("#").strip()
           if ln.startswith("## ") and stripped == header:
               start = i + 1
               break
       if start is None:
           return ""
       end = len(lines)
       for j in range(start, len(lines)):
           if lines[j].startswith("## "):
               end = j
               break
       return "\n".join(lines[start:end]).strip("\n")

   def list_op_inputs(group_info: dict) -> list[OpDataRef]:
       agents = group_info.get("agents") or []
       op = group_info.get("operation", "")
       if not agents:
           return []
       section = _OP_INPUT_SECTION.get(op)
       # Use only the first agent — siblings share the same input.md
       # structure; the user's mandate appears once per group.
       return [OpDataRef("agent_input", agents[0], section=section)]

   def list_op_outputs(group_info: dict) -> list[OpDataRef]:
       return [OpDataRef("agent_output", a)
               for a in group_info.get("agents") or []]

   def list_op_logs(group_info: dict) -> list[OpDataRef]:
       return [OpDataRef("agent_log", a)
               for a in group_info.get("agents") or []]

   def list_op_definition(group_info: dict) -> list[OpDataRef]:
       refs: list[OpDataRef] = []
       head = group_info.get("head_at_creation")
       if head:
           refs.append(OpDataRef("node_metadata", head))
       for nid in group_info.get("nodes_created") or []:
           refs.append(OpDataRef("node_metadata", nid))
       return refs
   ```

2. Update `.aitask-scripts/brainstorm/__init__.py` to re-export the
   public symbols.

## Verification Steps

1. Add `tests/test_brainstorm_op_refs.py` with unit tests for:
   - `OpDataRef` rejects an unknown `kind`.
   - `file_for_ref` resolves each kind to the correct path.
   - `_extract_md_section` correctly extracts a `## Header` subsection
     and returns "" for missing headers.
   - `list_op_inputs` returns one ref per group (using the first
     agent), with the correct section per op type.
   - `list_op_outputs` and `list_op_logs` return one ref per agent.
   - `list_op_definition` returns refs to head_at_creation and
     nodes_created nodes.
2. Run: `python -m pytest tests/test_brainstorm_op_refs.py -v`.

## Notes for Sibling Tasks

- The screen uses `list_op_inputs(group_info)[0]` to fetch the
  single canonical user-input ref; treat empty list (no agents
  registered yet) as "no input recorded yet, agent registration
  pending".
- `_extract_md_section` is intentionally minimal (`## ` heading
  match by literal string, no trim of trailing `#`). Do not extend
  it to support other heading levels — input.md files always emit
  user params under a top-level `## Header`.
- `_OP_INPUT_SECTION` is the single source of truth. Adding a new
  operation type means updating this dict and updating
  `list_op_inputs` callers — do not duplicate the section-name map
  elsewhere.
