## Context
This is the first child task of t176 (Create Web Site). The aitasks project needs a documentation website built with Hugo and the Docsy theme, deployed to GitHub Pages. This task sets up the foundational Hugo site scaffold that all other child tasks build upon.

The project has documentation in `docs/` (7 markdown files) and `README.md` but no web presence. We need to create a Hugo site in a `website/` subdirectory using the Docsy theme via Hugo Modules.

**GitHub repo:** https://github.com/beyondeye/aitasks
**Target site URL:** https://beyondeye.github.io/aitasks/

## Key Files to Create

1. **`website/hugo.toml`** — Full site configuration with Docsy module import
2. **`website/go.mod`** — Hugo module init (auto-generated by `hugo mod init`)
3. **`website/go.sum`** — Module checksums (auto-generated by `hugo mod get`)
4. **`website/content/_index.md`** — Minimal landing page placeholder
5. **`website/content/docs/_index.md`** — Docs section landing page
6. **`website/assets/scss/_variables_project.scss`** — Empty placeholder (required by Docsy)
7. **`website/assets/scss/_styles_project.scss`** — Empty placeholder (required by Docsy)

## Reference Files for Patterns

- `docs/` directory — contains the 7 markdown docs that will be migrated in t176_2
- `.github/workflows/release.yml` — existing workflow pattern
- `README.md` — project description to reference on landing page

## Implementation Plan

### Step 1: Install Hugo (if not already installed)
Verify Hugo extended edition is available. If not, install it. Hugo extended is REQUIRED for Docsy (SCSS processing).

### Step 2: Create website directory and initialize Hugo site
```bash
mkdir -p website
cd website
hugo new site . --force  # --force since directory exists
```

### Step 3: Initialize Hugo Modules
```bash
cd website
hugo mod init github.com/beyondeye/aitasks/website
```

### Step 4: Create hugo.toml
Replace the auto-generated config with a complete Docsy configuration:

```toml
baseURL = "https://beyondeye.github.io/aitasks/"
title = "aitasks"
contentDir = "content"
defaultContentLanguage = "en"
defaultContentLanguageInSubdir = false
enableMissingTranslationPlaceholders = true
enableGitInfo = true

[module]
  proxy = "direct"
  [module.hugoVersion]
    extended = true
    min = "0.155.3"
  [[module.imports]]
    path = "github.com/google/docsy"
    disable = false

[params]
  github_repo = "https://github.com/beyondeye/aitasks"
  github_project_repo = "https://github.com/beyondeye/aitasks"
  offlineSearch = true

[params.ui]
  sidebar_menu_compact = true
  breadcrumb_disable = false
  navbar_logo = true

[markup.goldmark.renderer]
  unsafe = true

[markup.highlight]
  style = "tango"

[outputs]
  home = ["HTML"]
  section = ["HTML"]
```

### Step 5: Download Docsy module
```bash
cd website
hugo mod get github.com/google/docsy@v0.10.0
hugo mod tidy
```
This creates/updates `go.mod` and `go.sum`.

### Step 6: Create placeholder SCSS files
```bash
mkdir -p website/assets/scss
touch website/assets/scss/_variables_project.scss
touch website/assets/scss/_styles_project.scss
```
These empty files are REQUIRED — Docsy's SCSS compilation fails without them.

### Step 7: Create minimal content pages

`website/content/_index.md`:
```markdown
---
title: "aitasks"
linkTitle: "aitasks"
---

{{< blocks/cover title="aitasks" image_anchor="top" height="med" >}}
<p class="lead mt-4">AI-powered task management for Claude Code</p>
{{< /blocks/cover >}}
```

`website/content/docs/_index.md`:
```markdown
---
title: "Documentation"
linkTitle: "Documentation"
weight: 20
menu:
  main:
    weight: 20
---

Documentation for the aitasks framework.
```

### Step 8: Add website/public/ to .gitignore
Hugo generates static files to `website/public/`. Add it to `.gitignore`:
```
website/public/
website/resources/
```

### Step 9: Verify
```bash
cd website
hugo server
```
Confirm the site starts without errors and is accessible at `localhost:1313/aitasks/`.

## Verification Steps

1. `cd website && hugo server` starts without errors
2. Browser shows landing page at `localhost:1313/aitasks/`
3. "Documentation" link in navbar works and shows the docs section
4. `go.mod` and `go.sum` exist and are committed
5. No SCSS compilation errors in Hugo output
